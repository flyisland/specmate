# docs/prd/

Product Requirements Documents.

## PRD lifecycle

```
draft -> approved -> obsolete
```

| Status | Meaning |
|---|---|
| `draft` | Being written. Not ready for design or implementation. |
| `approved` | Signed off. Design Docs and Exec Plans can reference this PRD. |
| `obsolete` | Feature killed or direction cancelled. |

## Naming

`prd-001-<slug>.md` — three-digit ID, globally incremented.
